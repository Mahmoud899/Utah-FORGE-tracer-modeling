# Derivative-Based Optimization Assessment

## Short Answer

Derivative-based optimization methods are plausible as **local refiners**, but they are **not a good full replacement** for the current global differential-evolution exploration.

## Trace of the Current Objective

The optimization objective is defined in:

- `two_components/problem_setup.py`

1. `objective_vector(x, t_true, c_true)`
2. `modelRELAP2(...)`

## What the Model Is Doing

The two-component model is:

`C(t) = fr1 * c1(t) + fr2 * c2(t)`

where each component curve is generated by the RELAP Laplace-domain model and then numerically inverted to time domain with the de Hoog inverse Laplace algorithm.

In the current `two_components/problem_setup.py`, the fitted parameters are:

- `MRT1`
- `MRT2`
- `Pe1`
- `Pe2`
- `fr1`
- `fr2`

The dual-porosity parameter itself is fixed to zero in the current setup, so this is simpler than a fully general dual-porosity fit.

## Would Derivative-Based Methods Work?

### Mathematically

Mostly yes, in the interior of the feasible region.

The forward model is built from smooth operations in the Laplace domain:

- exponentials
- square roots
- products
- ratios

and the de Hoog inversion uses a fixed contour and deterministic algebra for a given parameter vector.

So the forward map is generally smooth enough to support local derivative-based methods.

### Numerically

Only with some caution.

The de Hoog implementation contains numerically conditional steps, including zero-denominator guards and remainder logic. Those do not automatically make the objective unusable, but they can introduce numerical roughness.

Also, the current optimization has active constraints:

- `fr1 + fr2 <= 1`
- `MRT2 >= MRT1`

Those introduce boundary kinks where gradient-based algorithms can be less reliable.

## Practical Conclusion

### Good use case

Use derivative-based methods as a **local polish step** after a good initial guess has already been found.

Reasonable candidates:

- `SLSQP`
- `trust-constr`

### Bad use case

Do not rely on a pure derivative-based method to replace global exploration over the full parameter bounds.

Reason:

- the problem is still nonconvex
- multiple basins exist
- local methods are likely to get trapped in one basin

## Important Structural Observation

The mixture fractions enter linearly:

`C(t) = fr1 * c1(t) + fr2 * c2(t)`

So the difficult nonlinear part is mainly in:

- `MRT1`
- `MRT2`
- `Pe1`
- `Pe2`

while the fraction variables are much better behaved.

That suggests a useful hybrid strategy:

1. global or basin search on the transport parameters
2. local solve or local polish for `fr1` and `fr2`

## Empirical Note

A small local perturbation check around the current refined solution showed smooth changes in the objective for small parameter perturbations, without jumps or NaNs.

That supports the idea that local polishing is viable.

One caveat from that check: the recomputed objective did not exactly match the stored `best_J`, which suggests the current `data.csv` may not be the exact data snapshot used when that saved exploitation output was produced.

## Recommendation

Use a hybrid workflow:

1. global exploration with DE
2. basin identification
3. local derivative-based polishing within each basin

So the recommendation is:

- **yes** to derivative-based optimization for local refinement
- **no** to derivative-based optimization as the only optimizer for the full search
